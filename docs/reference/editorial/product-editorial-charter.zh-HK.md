**狀態：** 草稿，供產品負責人審閱

**英文版本：** 開發用翻譯；兩個版本如有歧義，以本版本為準  
**決定記錄截至：** 2026年7月11日  
**範圍：** 產品定位、報道範圍、證據、寫作、視覺、自主發布邊界及問責  
**相關規格套件：** [`../../specs/editorial-automation/`](../../specs/editorial-automation/)  
**文件角色：** 本章程記錄擬議中的未來產品，以及自動化新聞室可以運作的政策邊界。它是參考資料，不是實作指令；獲接納的規格會把所採納的原則轉化成具規範性、可測試的要求。本文件亦不表示現有 Discord 新聞室已實行這些規則。

## 1. 產品定義

首階段產品是一個為居英港人及家庭而設的新聞 app。內容以香港繁體中文撰寫，語氣自然貼近粵語。公開新聞網站不屬首階段產品範圍。

這個 app 協助讀者掌握：

- 英國生活及公共事務；
- 影響讀者在香港的家人、資產及決定的新聞；以及
- 對英國、香港，或與兩地任何一方有聯繫的家庭造成實質影響的國際事件。

我們對讀者的承諾是：

> 一個 app，讓你了解英國生活、香港時事，以及對英國、香港或與兩地任何一方有聯繫的家庭造成實質影響的國際事件。

產品以準確、實用及證據為本的報道為先，不以搶先或大量發布為目標。

產品不是翻譯服務。系統會把已發布的新聞當作線索，再回到公開證據及資料，並以香港語體撰寫原創繁體中文報道。

產品亦不是調查新聞室。系統不會訪問目擊者、追查私人文件、徵求洩密資料或調查指控；工作只建基於公開而可核實的材料。

目標營運模式是一個自主運作的 agentic 新聞室。符合資格的報道可由自動化系統完成發現、核實、草擬、檢查及發布，而毋須每篇例行等候人手批准。人類責任集中於政策、版本發布、例外個案、投訴、更正及緊急控制層面。自主運作必須受版本化規則、證據門檻、權利檢查、風險控制、可審計決定及失敗時關閉（fail closed）所限制。

## 2. 核心編採及自動化模式

每篇發布的報道均依循同一套基本模式：

1. 從官方渠道、具公信力的傳媒或其他獲准使用的公開來源，發現可能值得報道的事件。
2. 以具權威性的文件、資料或可靠的已發布報道，建立可追溯的證據包。
3. 分辨哪些內容已獲確認、權威來源有否明確標示任何資料屬暫定，以及相較較早報道有甚麼改變。
4. 撰寫原創報道，而不是翻譯或貼近原文改寫來源文章。
5. 只有當分析已由具名而可信的來源發布，並有可追溯證據支持時，方可採用該分析。
6. 執行所有必需的證據、原創性、權利、視覺、敏感內容、法律風險、資料格式及呈現檢查。
7. 作出三種發布決定之一：自動發布、暫緩並交由獲授權人員覆核，或拒絕發布。
8. 記錄消息來源、核心說法與證據的連結、權利狀況、模型及 prompt 版本、驗證結果、風險標記、發布決定、發布時間，以及其後任何更正或撤稿。

只有在所有必需門檻均通過，而且沒有任何強制暫緩條件時，報道方可自動發布。目標營運模式不要求每篇報道例行由人手批准；只有當獲接納的規格把某項例外列為可覆核，而不是必須自動拒絕時，方須交由人類覆核。

服務只發布已確認的事實，不會把傳聞、匿名說法、社交媒體反應或模型生成的推論當作新聞。任何模型或 agent 均不得豁免門檻、修改管治政策、把逾時當作批准，或為達到發布配額而發布內容。

## 3. 地理範圍

每篇報道均設有一個或以上地理標籤。標籤指出事件發生地或作出決定的地方，以及證據顯示受到實質影響的其他地方。

例如，英國一項改變 BN(O) 身份安排的決定，可同時標示為英國及香港。讀者篩選任何一個地理範圍時，都會看到該報道。

### 3.1 英國

英國報道包括全國及地方新聞。英國報道可包含三個層級的地理資料：

1. 全英國；
2. 英格蘭、蘇格蘭、威爾斯或北愛爾蘭；以及
3. 相關城市、郡、自治市鎮、地方議會轄區或其他具實際意義的地區。

地點只會細分至證據所支持的最具體層級。系統不得自行推斷更精確的位置。

### 3.2 香港

香港是一個完整的地理分區，不會再按地區細分作篩選或導覽。

香港新聞毋須證明對英國有直接影響。對家人、物業、財務或未來決定仍與香港有聯繫的讀者而言，香港新聞本身已有價值。

### 3.3 全球

全球是一個完整的地理分區，不會再按國家或洲分拆。

一則國際新聞一般只有在符合以下至少一項情況時，才納入報道：

- 影響英國或香港的安全；
- 影響能源、糧食、旅遊、供應鏈、貨幣、利率或家庭生活成本；
- 改變英國或香港政策；
- 影響兩地之間的往來或聯繫；
- 為與兩地均有聯繫的家庭帶來實質風險；或
- 屬具有非凡國際公共重要性的事件。

如有證據支持，報道應自然解釋有關影響，但不得每篇都加入公式化、重複的「這件事為何與你有關」段落。

## 4. 內容分類

每篇報道除地理資料外，至少設有一個內容分類。當同一篇報道確實橫跨多個範疇時，可歸入多個分類；但不會因文章順帶提及某個議題，就加上相關分類。

| 分類 | 一般報道範圍 |
|---|---|
| 政治及法律 | 政府決定、立法、選舉、法院及公民權利 |
| 入境及身份 | BN(O)、簽證、定居、公民身份、庇護及非正常移民政策 |
| 安全及罪案 | 已核實事故、警方警告、檢控、法院結果及公共安全 |
| 天氣及災害 | 風暴、水浸、極端天氣、火災及環境危害 |
| 交通及基建 | 具實質影響的中斷、罷工、封閉、重大工程及政策改變，包括主要英港航空、機場或空域中斷、航線改動、旅遊規則改變及乘客權益發展 |
| 健康及醫療 | 公共衞生、疫情、英國國民保健服務（NHS）及香港醫院管理局政策、藥物及服務改變 |
| 教育及校園 | 由幼兒教育至大學，包括收生、考試、停課、校園安全及政策 |
| 稅務及福利 | 稅項、福利、退休金、家庭支援及官方限期 |
| 工作及就業 | 僱傭權利、罷工、大規模裁員、勞工政策及求職騙案 |
| 房屋及本地生活 | 租金、按揭、房屋政策、樓宇安全、規劃及公用服務 |
| 經濟及金融 | 通脹、利率、貨幣、重大市場事件及對家庭的影響 |
| 消費者權益及騙案 | 產品回收、服務失靈、詐騙、退款、資料外洩及消費者保障 |
| 科技及網絡安全 | 大規模服務中斷、網絡事故、科技政策及安全 |
| 戰爭及國際事務 | 戰爭、制裁、外交、供應鏈及重大國際風險 |
| 社區及公共服務 | 地方議會服務、諮詢、支援計劃及對社區有實質影響的改變 |

### 4.1 明確不納入的內容

產品不會報道：

- 娛樂、名人新聞或八卦；
- 一般體育賽果、轉會或球會新聞；
- 餐廳、購物、酒店或生活品味推介；
- 產品集合推介、折扣碼或聯盟行銷式內容；
- 一般活動列表或商業社區宣傳；
- 個別航班、列車或道路的一般延誤；
- 實時航班追蹤或一般交通狀況服務；
- 與英國、香港無實質關聯，亦不具特殊公共利益的國際新聞；或
- 評論專欄、社論或政黨政治倡議。

娛樂或體育活動如引致具實質影響的交通、安全或公共服務問題，仍可報道；但會歸入相應的公共影響分類，而不是當作娛樂或體育新聞處理。

## 5. 新聞價值及發布量

監察工作可以每小時進行，但不設每小時、每日或每個分類的發布配額。

只有當一則新聞具備足夠證據及實質新資料時，才會發布。產品不得為維持活躍度、平衡分類或達到字數目標而製造填充內容。

交通、航空、天氣及公用服務事故，只有在影響相當數量的人、持續一段具實質意義的時間，或明顯影響日常生活時，才構成新聞。這個 app 不會照搬每一則服務狀態更新。

供內部揀選新聞時，一般要由已核實事實證明至少符合以下一項：

- 法例、權利、身份、官方限期或公共政策有所改變；
- 對安全或公共衞生有可信而實質的影響；
- 對必要服務、路線、學校、工作場所或地區造成重大中斷；
- 對家庭財務、工作、房屋、教育、醫療或英港往來造成實際影響；
- 存在受影響讀者可能需要採取行動的官方限期、指示或程序；或
- 在香港或國際層面具有非凡公共重要性。

這是新聞揀選準則，不是向讀者展示的緊急程度評分。一般航班延誤、短暫而孤立的道路封閉，或沒有上述任何影響的一般外國政治聲明，均不符合要求。

首階段產品不設固定早報、晚報、每日十大新聞或熱門排行榜。

## 6. 證據及消息來源政策

### 6.1 消息來源分級

消息來源分為四級評估：

1. **第一手及官方證據：** 法例、法院判決、政府文件、官方統計、警方及緊急服務部門聲明、學校或交通通告、監管申報文件及公司公告。
2. **具公信力的新聞機構：** 有清晰編輯流程、消息來源及更正機制的報道。
3. **地方及專業刊物：** 可用於地方或專門範疇報道；如說法涉及重大事項，須有額外佐證。
4. **只可作線索的來源：** 社交媒體、討論區、網誌、匿名網站、通訊群組及沒有註明出處的截圖。

第四級來源可以揭示一則可能的新聞，但不能證明事件屬實。

### 6.2 證據門檻

如有具權威性的第一手證據，文章須直接使用，並交代在何處找到。例子包括：

- 以英國稅務海關總署（HMRC）資料核實稅務規則；
- 以《入境規則》（Immigration Rules）或英國內政部指引核實入境政策；
- 以英國國家統計局（ONS）資料核實官方統計；
- 以英國氣象局（Met Office）資料核實英國天氣警告；
- 以學校或地方議會通告核實停課或關閉安排；
- 以 National Rail 或營運商資料核實具實質影響的鐵路中斷；
- 以已發布的判詞核實法院裁決；以及
- 以香港法例、憲報公告或政府部門刊物核實香港政策。

如找不到公開而可靠的證據，系統須拒絕該項目；只有在獲授權覆核人員能處理某項明確不確定因素時，方可暫緩處理。系統不會聯絡目擊者、不會要求家庭提供私人文件，亦不會以調查記者方式繼續追查。

最低證據門檻視乎說法類型而定：

| 說法類型 | 最低發布依據 |
|---|---|
| 官方規則、決定或限期 | 現行的第一手文件可證明負責機構作出了甚麼決定、發布了甚麼內容，或令甚麼措施生效。 |
| 官方數字或資料集 | 原始發布文件、資料集、表格或新聞稿，可證明其所述參考期內的相關數字。 |
| 發展中的事故或服務中斷 | 由具權威性的機構或營運商確認可觀察到的事實。如沒有合適的第一手來源，至少須有兩個互相獨立而可靠的已發布來源，佐證同一項範圍明確的事實。 |
| 其他事實事件 | 由直接負責的權威公共來源確認該項範圍明確的事實，或至少由兩個互相獨立而可靠的已發布來源共同佐證。 |
| 嚴重指控、身份、動機或刑事責任 | 必須同時具備合適、公開而可核實的證據，以及已記錄的誹謗、私隱及法庭風險評估。缺少任何一項，均不得發布有關指控。 |
| 分析、模型或預測 | 必須由具名、可信而可追溯的來源發布該項分析及其證據基礎。 |

多間傳媒重複同一宗原始報道，不算互相獨立的佐證。

### 6.3 官方事實與官方意見

官方來源可以證明該機構作出了甚麼決定、做過甚麼、記錄了甚麼或公開說過甚麼，亦可證明規則何時生效、文件載有甚麼內容，或官方數據系列報告了甚麼。

但官方來源不會自動證明涉及另一人的指控、具爭議的行為、因果關係、罪責，或該機構對成效及責任歸屬的自身評價。警方聲明可以證明某人被捕或被檢控，但不能證明有罪。已公布的計劃成本可以是事實；至於部長聲稱計劃成功，如無證據支持，仍只屬該名部長的評價。

一項聲明不會僅因由官員作出，就自動成為獨立新聞。它必須記錄一項決定、造成實質公共後果，或另具獨立新聞價值。標題或導言不得把未經證實的官方指控，當成無保留的事實重述。

### 6.4 匿名消息來源

如核心事實只由未具名官員、匿名消息人士、洩密資料或無法核實的截圖支持，不得當作已確認新聞發布。

即使註明說法來自具公信力的新聞機構，亦不會令匿名或無法核實的指控變成可發布內容。重複指控可能重複造成同一法律及事實傷害。這個 app 會等待公開而可核實的證據，才報道背後的實質說法。

### 6.5 資料

資料報道必須按適用情況列明資料提供者、刊物、參考期、相關資料集、表格或章節，以及直接來源連結。

只有當同一個權威來源本身已採用一致定義發布比較，文章才可重現該比較。根據本章程，系統不得自行計算新的比較結果、比率、比例、排名或人均數字。

如定義曾經改變，改變前後的數字不得當作可直接比較。

### 6.6 社交媒體

官方社交媒體帳戶可視作官方聲明；其他社交帖文只可作線索。

產品不會以「網民震怒」、少量留言、讚好數目或沒有註明來源的影片作為文章基礎。正式民調可按其已發布結果及方法報道，但不得作出超越民調本身所能證明的推論。

### 6.7 文章內的消息來源標示

一項事實首次出現時，須清楚交代消息來源。其後如連續數項事實均來自同一個已清楚標示的來源，毋須每句重複註明。

具爭議的說法、分析、估算、預測，以及機構對自身表現的評價，均須逐項標明出處。完整來源連結會保留在文章頁尾。

## 7. 分析政策

這個 app 不會自行創作分析。

只有當分析可追溯至以下來源時，方可納入：

- 官方或監管機構評估；
- 具名專家；
- 學術研究；
- 已發布的機構模型；
- 具公信力的市場或行業分析；或
- 來源本身用以支持結論的歷史資料。

分析員或機構必須具備相關專業知識，而其方法、證據或資料亦須有足夠可追溯性，讓系統或獲授權覆核人員評估有關說法。如明顯存在重大的商業、政治或倡議利益，文章須加以交代。單憑「專家」、「分析員」或「智庫」等名銜並不足夠。

文章須交代由誰作出分析、採用了甚麼證據，以及任何已列明的條件。例如：

> 路透社報道，引述具名能源分析員指出，若航運中斷持續，可能推高煉油及運輸成本。

這個 app 可以用清晰中文解釋有關分析，但不得把它包裝成 app 自身的預測。如沒有可信而已發布的分析，文章只報道事實，並省略推測性影響。

不得推斷身份、動機、群體行為或刑事責任。只有在合適的官方來源或法院確立後，方可報道。

## 8. 原創性及版權

## 9. 視覺政策

視覺素材的優先次序如下：

1. 具有效編採用途牌照或明確重用許可、並與事件直接相關的真實圖片；
2. 只以已核實材料為基礎、獨立製作的事實資訊圖；以及
3. 如上述兩者均不合適，使用統一品牌設計的新聞卡。

### 9.1 真實照片

只有在 app 能記錄以下資料時，方可使用事件照片：

- 權利持有人或獲授權的許可方；
- 牌照或許可內容；
- 獲准使用的平台、地區及期限；
- 必須列出的署名；以及
- 任何裁剪、編輯或重用限制。

版權許可與圖片所涉人物及內容的風險審查是兩回事。發布前，系統必須完成適用檢查，判斷圖片有否披露受保護身份、兒童、受害人、正接受治療的人、私人資料、車牌，或其他可能引起私隱、資料保障、報道限制或藐視法庭風險的細節。如檢查結果不確定，便不得自動發布。

不得假設政府或警方網頁上的圖片可供重用。相關頁面必須明確把圖片置於適用牌照之下，並須遵守任何第三方署名要求或不在授權範圍內的標誌限制。

這個 app 不得把未獲授權的 BBC、通訊社、社交媒體或其他出版商照片上載至圖像模型，再轉成卡通、Q版圖片、畫作、描摹、重新着色或其他衍生作品。

這是一項嚴格的產品規則，並非聲稱每一種風格轉換都必然違法。上載來源照片本身可能已製作或傳送未獲授權的複本；如輸出重現原照片構圖或表達中在質量上屬實質的部分，亦可能另行構成侵權。

### 9.3 事實資訊圖

資訊圖用來呈現已核實事實，不會假裝是事件現場照片。內容可包括：

- 路線圖或位置圖；
- 受影響的交匯處、路線或地區；
- 時間線；
- 官方數字；
- 新舊政策比較；
- 服務改動；或
- 有列明來源的圖表。

每項事實元素均須由已列出的來源支持。資訊圖須標示來源資料的時間，並附上明確說明，例如：

> 事故資訊圖，根據警方及 National Highways 資料製作。並非事故現場圖片。

資訊圖不得虛構車輛型號、顏色、救護車數目、天氣情況、傷勢或成因。

簡單、獨立繪製或已妥善取得牌照的圖示，可代表一項已核實事實，例如道路封閉、救護車到場或天氣警告。圖示不得重組事故現場、暗示沒有來源支持的空間配置，或模仿來源照片的構圖。底圖、來源圖表及底層資料集亦須具備合適的重用權；團隊不得描摹受專有權保護的地圖或圖表。

### 9.4 資料圖片

如一張已獲授權的通用或資料圖片確實有助提供背景，可予使用。圖片必須清楚標示為資料圖片，亦不得令人誤以為它拍攝了所報道事件。

### 9.5 禁用視覺素材

產品不會使用：

- 未獲授權的出版商圖片、縮圖或截圖；
- 以寫實或卡通方式重構意外、罪案、戰爭或災害現場；

- 事實細節無法追溯來源的視覺素材。

## 10. 寫作及呈現方式

### 10.1 語言

文章使用香港書面語體的繁體中文，並在合適情況下採用自然粵語語氣。嚴肅新聞不會為口語化而勉強使用生硬或過度口語的寫法。

首次提及讀者未必熟悉的英國機構或政策時，須提供清晰中文解釋，並保留官方英文名稱或縮寫，例如：

> 永久居留（Indefinite Leave to Remain，ILR）

如沒有公認的中文譯名，應保留官方英文用語，而不是自行創作譯名。

### 10.2 中立原則

這個 app 不偏袒任何政黨。它不會支持政黨或候選人、不會發布社論，亦不會把意見包裝成新聞。

中立不代表要淡化已證實的事實，亦不代表要把沒有證據支持的說法視作同等可信。已確認事實須清楚陳述；說法、建議及意見則須列明來源。

涉及政治敏感的地名、機構及身份，須採用香港通行而中立的用語及官方名稱。帶有立場的標籤及政治口號，只可在其用法本身具有新聞意義時出現，並須註明由誰使用。

### 10.3 標題

標題須交代地點、事件及最重要的已確認進展。標題不得：

- 使用標題黨手法或煽情誘餌；
- 暗示未獲確認的成因；
- 把可能性寫成確定事實；
- 把被捕或被檢控寫成已定罪；或
- 在入境身份、國籍或身份與事件沒有實質關係，或未獲官方確認時，把它加入標題。

### 10.4 長度及結構

文章不設固定字數。地方服務中斷可能只需數段；複雜的稅務、入境或國際新聞，則可能需要較長解釋。

當讀者已有足夠已核實資料理解事件，文章便應停止。不得為達到目標字數而加入空泛結論或填充內容。

以下項目只屬內部編採檢查清單，並非文章必須顯示的標題：

- 發生了甚麼事；
- 哪些內容已獲確認；
- 權威來源有否明確表示任何已發布數字或狀況屬暫定；
- 有甚麼改變；
- 有否具來源支持、對英國或香港讀者造成的影響；以及
- 有否重要的官方行動或限期。

### 10.5 日期及時間

事件採用事發地的當地時間。重要跨境限期可同時列出英國及香港時間。如有機會混淆，須註明格林威治標準時間（GMT）、英國夏令時間（BST）或香港時間（HKT）。

如資料需要在存檔中長期保持清晰，應優先使用完整日期，而不是「今日」等相對時間說法。

## 11. 敏感題材報道

### 11.1 姓名及個人資料

如正式來源已確認身份及職銜、身份與事件相關，而且不受法律限制，公職人員或獲授權的機構代表，可因其官方公開行為或聲明而被點名報道。

對任何其他人士，編採上可考慮披露身份的四項已議定條件為：

1. 可靠而正式的來源已發布該身份；
2. 該身份與報道有直接關係；
3. 系統或獲授權覆核人員已確認資料所指為同一人；
4. 沒有法庭命令、自動生效的報道限制或其他法律保障禁止辨識該人。

符合上述四項條件，本身並不等於已取得合法發布依據。如涉及私人個體，或任何因指控、受害人身份、涉嫌犯罪、敏感個人資料或非公開行為而被點名的人，發布決定紀錄亦須說明：為何披露身份屬必要及合乎比例、可能造成的傷害、任何合理私隱期望、是否有較少侵擾的替代做法，以及所涉的特殊類別個人資料或刑事罪行資料。

資料已公開，並不會自動免除私隱、準確性或法庭報道責任。

### 11.2 兒童及學校

報道範圍可涵蓋幼兒教育、學校、專上學院及大學。這個 app 會報道已核實的關閉或停課、疫情、安全事故、收生、考試、費用、罷工及政策改變。

產品不會辨識受保護兒童、不會重複家長群組傳聞，亦不會發布涉及學生或教職員的未核實指控。

### 11.3 罪案

被捕、被檢控、審訊、定罪及判刑是不同階段；每個階段均須使用正確字眼。

不得推斷種族、宗教、國籍、入境身份或動機。只有在官方確立且與事件有實質關係時，方可提及。

任何報道如涉及現有或擬展開的法律程序，或受保護的法庭材料，系統在發布或更新前都必須按獲接納規格完成當時適用的司法管轄區檢查。如檢查不能得出可靠結果，報道須暫緩並交由獲授權人員覆核。這不只適用於被捕、檢控、審訊及判刑報道，亦包括刑事及民事程序、家事案件、上訴及死因研訊。檢查範圍包括：

- 法律程序是否已處於進行中；
- 相關司法管轄區內自動生效或由法院頒令的報道限制；
- 受保護的申訴人、證人、兒童及家事程序；
- 延期報道令或匿名令；
- 多項細節合併後造成的拼圖式識別；以及
- 會否妨礙現有案件或相關程序的公平審理。

單憑搜尋不到具名匿名令並不足夠。如無法確定適用的限制或法律程序是否已處於進行中，須刪去可識別人士或可能妨礙審訊的細節；有需要時，整篇報道須暫緩發布。

### 11.4 健康

這個 app 會報道公共衞生證據、服務改變及有列明來源的醫學分析。它不會為讀者作診斷、不會指示讀者停止或更改治療，亦不會把小型初步研究說成已證實的療法。

### 11.5 金融

這個 app 會報道經濟政策、家庭財務及重大市場影響。它不會提供股票貼士、目標價、個人化投資指示、加密貨幣推廣或聯盟行銷式金融內容。

### 11.6 戰爭

交戰一方提出的說法，仍須明確標示為該方說法。傷亡數字須沿用來源本身的分類，並列明來源及報道時間。如來源有區分平民與戰鬥人員，或死亡與受傷，文章須保留該區分；如來源沒有區分，文章須如實說明，不得自行估算比例。

這個 app 不會發布可能危及他人的行動細節、強烈血腥圖片、把宣傳當作事實，或在沒有證據支持下預測勝利、崩潰或戰事結束日期。

### 11.7 自殺及自殘

只有在具清晰公共重要性時，方會報道。內容須避免交代方法細節、作過度簡化的解釋、浪漫化事件、刊載遺書，或不必要地辨識當事人及其家人。

## 12. 文章生命週期及 app 呈現方式

### 12.1 動態消息及篩選

主動態消息會顯示所有已發布報道，並按首次發布時間倒序排列。系統不會按熱門程度、點擊量、推斷偏好或編輯重要性排名。

讀者可按以下條件篩選：

- 英國、香港或全球；
- 適用時按英國地方範圍；以及
- 一個或以上內容分類。

編採系統不會設定紅色、琥珀色或其他緊急程度級別。

所有已發布報道，不論通知設定如何，均會保留在 app 內。通知屬自選功能，只設簡單開／關控制。啟用後，通知會跟隨讀者選擇的地理範圍。產品不設獨立緊急通知模式，亦不設按緊急程度分級的通知。

### 12.2 最新發展及相關報道

如有具實質意義的新進展，應另發一篇新文章。原有文章不會為吸收每宗後續事件而不斷重寫。

如進展加入新近確認的決定、規則、限期、官方結論、檢控、判決、可量度改變或具實質意義的事故結果，即屬重大進展。新文章須以該進展作開首，只重述理解事件所需的背景。

關於同一事件、案件、政策、法案或正式程序的文章，須互相連結為相關報道。只有共同關鍵字或同屬廣泛分類並不足夠。例如，同一案件的被捕、檢控及判決互有關聯；兩宗互不相關的入境新聞則沒有。

### 12.3 文章元資料及問責

文章頁首顯示：

- 標題；
- 首次發布時間；
- 如適用，最後更新時間；
- 一個或以上地理範圍；
- 一個或以上內容分類；以及
- 負責出版者或自動化新聞室身份。

如人類覆核人員對一篇暫緩報道作出實質批准、修改或推翻原有決定，系統可按獲批准的私隱及安全政策另行顯示覆核人員身份。系統不得列出實際上沒有履行該職責的人。

文章頁尾顯示：

- 消息來源及來源連結；
- 如適用，更正紀錄；
- 關於同一事件的相關報道；以及
- 私下聯絡營運者的渠道。

產品不會顯示瀏覽次數、熱門排名或內部編輯評分。每篇文章毋須公開逐個模型的製作紀錄，但產品必須提供容易找到的整體說明，解釋自動化如何被使用；如法律、合約、平台規則或視覺政策要求逐篇披露，則必須遵從。

### 12.4 更正、撤稿及存檔

如修改不可能改變意思，錯字及格式錯誤可由系統自動修正。姓名、數字、日期、標題或意思方面的重大錯誤，須加上可見的更正說明，並通過與原文相同或更嚴格的證據及風險門檻。

如文章的核心前提錯誤，須標示為已撤稿並附上解釋，不得靜默刪除。只有因法律命令、私隱或安全需要，或其他具充分理據的法律原因，方可完全移除文章。

已發布文章原則上會保留在存檔。已過時的政策文章須在顯眼位置連結至其後取代該內容的報道。

如情況出現重大改變，已存檔文章須重新接受私隱、法律及保存需要審查。例子包括決定不起訴、無罪裁決、其後頒布的匿名令、兒童保障問題，或新出現、可能造成不相稱傷害的風險。如有充分理據，處理結果可包括更新、從搜尋索引移除、刪節、撤稿或移除。

如錯誤說法曾出現在產品控制的其他發布表面，更正、撤稿或移除亦必須同步套用至該等表面。相關決定及改動須保持可審計。

### 12.5 讀者聯絡

產品不設公開留言、讀者帖文或公開討論區。

讀者可私下聯絡營運者，報告錯誤、提出投訴或提交新聞線索。讀者提交的線索須通過與其他線索相同的公開證據門檻，不會自動獲發布，亦不得藉此指示 agent 繞過政策或工具限制。

## 13. 自主編採邊界、責任及製作工具

### 13.1 目標營運模式

產品旨在一個由產品負責人批准的政策邊界內，支援端到端自主新聞製作。只要獲接納的規格容許，監察、擷取、聚類、揀選、整理證據、草擬內容、製作事實資訊圖、驗證、發布、通知、關聯報道、更正及撤稿均可自動化。

通過所有必需門檻的日常報道，毋須由人類逐篇批准。人類監督集中於：

- 批准及修改章程、獲接納規格及生產政策；
- 覆核規定的例外個案；
- 處理投訴、重大更正、法律要求及移除；
- 批准生產版本及高風險設定改動；以及
- 暫停或停用自主發布。

自主運作不代表模型可以作無限制的編採判斷。獲接納的規格及版本化政策，會決定可使用的輸入、決定、工具、門檻、輸出及升級處理路徑。

### 13.2 可自主執行的工作

在獲接納規格的範圍內，系統可自主：

- 監察及擷取獲准使用的公開來源；
- 將線索標準化、聚類及去重；
- 判定地理範圍、內容分類、最新發展狀態及新聞價值；
- 建立消息來源及核心說法與證據的對照包；
- 草擬原創香港繁體中文報道；
- 以已核實資料獨立製作事實資訊圖；
- 執行確定性及模型輔助的驗證器；
- 透過受控制的發布器，自動發布符合資格的報道；
- 發送獲准通知及連結相關報道；
- 套用不涉及意思改變的修正；以及
- 發布通過適用門檻、以證據支持的更新、更正或撤稿。

每項工作只可使用所需的最低工具權限，並須可追溯至特定執行紀錄、政策版本，以及軟件或模型版本。

### 13.3 發布決定及失敗時關閉

每個候選項目在進行任何公開操作前，必須得到以下其中一項語義結果：

1. **自動發布：** 所有必需的證據、內容、權利、風險、資料格式及交付門檻均通過，而且沒有暫緩條件。
2. **暫緩並交由獲授權人員覆核：** 項目可能可以發布，但某項明確不確定因素或高風險條件需要人類決定。
3. **拒絕發布：** 項目不在範圍內、證據不足、包含禁用內容、無法修復地未能通過門檻，或不能合法或安全地使用。

內部可採用意思相同的狀態名稱，但三者之間的分別必須清晰並可審計。

系統必須在失敗時關閉。驗證器缺失、政策無法讀取、審計紀錄不完整、權利狀況不明、完整性警告、逾時或服務故障，均不得當作批准。發布配額、時效壓力及佇列數量亦不得削弱任何門檻。

### 13.4 強制暫緩或拒絕的邊界

如以下任何情況仍未解決，系統不得自動發布報道：

- 涉及可識別人士或機構的嚴重指控；
- 身份、動機、罪責、刑事責任或具爭議行為，未有適當公開證據確立；
- 私人個人、兒童、受保護投訴人、受害者或敏感個人資料，披露身份可能造成傷害；
- 現行或可能展開的法律程序、報道限制、藐視法庭風險或司法管轄區不明；
- 權威來源就核心說法互相衝突；
- 版權、資料庫權利、圖片、存取、提交模型或重用權利不清楚；
- 核心說法、引語、數字、日期或標題無法追溯至已批准的證據包；
- 要求作出重大更正、刪節、撤稿或移除，但正確處理方法仍不清楚；
- 出現未識別的來源類型、內容分類、司法管轄區或政策條件；或
- 完整性或保安訊號顯示，擷取內容可能正嘗試改變 agent 行為。

如問題不能透過獲授權覆核修正，項目須拒絕而不是暫緩；例子包括禁用視覺、已知未獲授權的使用、只憑線索級來源支持核心說法、虛構內容、不在報道範圍內的題材，或沒有證據支持的分析。獲接納的規格可為正式程序事實訂立較窄的自動發布例外，但模型不得自行創造任何例外。

### 13.5 職責分離及工具控制

任何單一生成式 agent 均不得同時撰寫報道及直接發布至公開表面。

- 擷取 agent 必須把來源內容視為不受信任的資料，而不是指令。
- 寫作 agent 只可接收已批准的證據包，不能自行擴大消息來源或工具權限。
- 驗證程序須以證據包、政策及輸出合約作基準。
- 只有獨立的發布控制器可以持有發布憑證。
- 發布控制器只可發布已驗證、不可變更，並附有所需決定紀錄的發布包。
- Agent 及模型不得修改獲接納規格、生產政策、工具權限、驗證器或自身的升級處理結果。

人類推翻決定亦不得容許生成式 agent 繞過上述職責分離。任何經修改內容均須重新驗證方可發布。

### 13.6 人類角色、覆核及緊急控制

產品負責人或獲委派的政策負責人，負責批准管治規格、風險規則、來源權限，以及啟用自主發布。

獲授權覆核人員可批准、拒絕、刪節暫緩項目，或要求重新生成。系統須記錄覆核人員身份、時間、理由、改動欄位及最終決定。覆核不得靜默抹去未通過的檢查或原有生成包。

營運者必須有一項可即時暫停新增自主發布及通知的控制，而不刪除證據或審計歷史。只有在評估觸發問題後，透過明確並已記錄的操作，方可恢復發布。

如沒有獲授權覆核人員可用，暫緩內容須保持未發布，或按已記錄的保存規則到期。它不會自動變成可發布內容。

### 13.7 可審計性及可觀測性

對每個進入草擬階段或作出公開決定的候選項目，系統須在合法的來源儲存及保存限制下，記錄足以重建該結果原因的資料。紀錄按適用情況包括：

- 消息來源識別資料、存取時間及來源權威分類；
- 支持每項核心說法的證據；
- 消息來源及素材權利狀況；
- 已抽取事實、暫定標記及重大衝突；
- 模型、prompt、template、政策、驗證器及軟件版本；
- 生成草稿及發布包的雜湊值；
- 驗證結果、修復、風險標記及升級處理原因；
- 最終決定，以及決定由自動化系統或獲授權覆核人員作出；
- 發布目標及時間；以及
- 其後更新、更正、撤稿、移除及投訴。

系統必須監察發布失敗、異常暫緩或拒絕比率、未獲證據支持說法的偵測、重複發布、更正頻率，以及政策或驗證器錯誤。如未能建立所需審計紀錄，便不得發布。

### 13.8 改動控制

生產 agent 不得自主修改本章程、獲接納規格或執行有關要求的政策。模型、prompt、驗證器、來源 adapter 及發布控制器的改動，必須版本化，並按獲接納的質素及改動控制規格，在投入生產前完成評估。

新模型或 prompt 版本不會因上一版本具有生產權限而自動繼承該權限。發布程序必須證明新版本仍符合獲接納要求，並可在需要時回復上一版本。

### 13.9 面向讀者的問責

產品須提供清晰而容易找到的說明，交代新聞室會使用自動化及生成式系統、該等系統獲准執行甚麼工作、甚麼情況會交由人類覆核、讀者如何報告錯誤，以及由誰營運服務。

每篇文章須列明負責出版者或自動化新聞室身份、消息來源、發布及更新時間，以及更正紀錄。只有在紀錄所示工作流程確實有人履行有關職責時，方可把該人列為覆核人員或作者。產品不得虛假聲稱由機器協助製作的內容完全由人撰寫或批准。

產品毋須向讀者公開內部 prompt、模型推理或完整製作日誌。如法律、平台規則、供應商條款、發布地區或視覺政策要求披露，該要求凌駕本呈現規則。

## 14. 法律風險取態

相比翻譯及重新發布來源文章，這套模式可實質降低風險，但不會免除出版者責任。自主發布只會改變由誰或甚麼系統作出操作決定，並不會把法律責任由營運者轉移出去。

英格蘭及威爾斯、蘇格蘭、北愛爾蘭、香港及其他發布地區的適用法律並不相同。誹謗、私隱、兒童保障及法庭報道檢查，必須依據相關司法管轄區的法律，而不能籠統假設全英國規則相同。

營運者仍須對以下事項負責：

- 版權及資料庫權利；
- 誹謗及惡意虛假陳述（malicious falsehood）；
- 私隱及資料保障；
- 法庭報道限制及藐視法庭；
- 照片、圖像、資訊饋送（feeds）及數據所附帶的權利及牌照條款；
- 自主發布控制的設計、驗證及監察；以及
- 準確的更正及下架決定。

根據本草稿，以下項目是產品推出的先決條件。它們必須寫入獲接納規格，並在啟用自主公開發布前由產品負責人批准：

1. 消息來源及素材權利登記冊；
2. 禁用來源及禁用方式清單；
3. 書面的更正、投訴及移除程序；
4. 包含可由機器執行的暫緩及拒絕條件之法庭、兒童、私隱及指控風險政策；
5. 失敗時關閉的發布控制器、完整決定審計、例外佇列及緊急停止功能；
6. 適用於模型、prompt、驗證器及來源 adapter 的版本化評估及改動控制程序；以及
7. 由具適當資格、熟悉傳媒法及版權法的律師，按英國各司法管轄區、香港及任何其他重要發布地區，使用具代表性的文章、自動化決定及視覺例子進行審查。

本章程是一套編採及自動化政策，並不保證任何特定報道、自動化決定或素材必然合法。

## 15. 實例：M4 車輛事故

假設具公信力的報道指，一輛車在 M4 翻側，緊急服務人員到場。

自主系統必須：

1. 透過警方、National Highways 或其他具權威性的公開材料，核實地點、時間、道路狀況及已確認傷亡；
2. 只發布上述來源所支持的細節；
3. 撰寫原創報道並列明消息來源；
4. 只有在取得並記錄合適的編採用途牌照後，才使用原有現場照片；
5. 絕不把未獲授權的出版商照片輸入圖像模型，以製作 Q版或其他衍生圖片；
6. 如沒有已獲授權的照片，製作有列明來源的路線圖、時間線或事故資訊卡，並清楚標示為資訊圖，而不是事故現場圖片；以及
7. 只有在所有證據、權利、內容及風險門檻均通過時，方可自動發布；否則須按明確原因拒絕該項目，或暫緩並交由獲授權人員覆核。

資訊圖可顯示受影響交匯處、方向、封閉情況、已確認的應變措施及更新時間。它不得虛構車輛類型、顏色、人物、救護車數目、天氣或成因。

這個例子用以說明自主邊界：如公共服務報道屬日常性質、消息來源充分而且所有門檻均通過，便可毋須逐篇人手批准；但如傷亡、身份、成因、圖片權利或法律風險仍有不確定因素，便不得自動發布。

## 16. 本章程不涵蓋的範圍

以下事項需要另行作出產品或工程決定，本章程刻意不作定案：

- 價格、收費、試用或訂閱設計；
- 帳戶建立及身份識別供應商（identity provider）的運作方式；
- App Store 及 Play Store 實作；
- 模型、供應商、agent（代理程式）或 SDK 選擇；
- 新聞室基建及託管；
- 詳細的資料擷取、儲存及檢索架構；
- 市場推廣、用戶獲取及收入規劃；以及
- 對現有程式碼儲存庫的實作改動；該等改動須另有獲接納規格及實作計劃。
